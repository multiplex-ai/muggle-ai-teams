## Non-Coding Specialist Output Format

- **Section**: Which part of the deliverable they own
- **Proposed content/structure**: Concrete outline (slide titles, email subjects, itinerary items)
- **Tone/style guidance**: How this section should read
- **Research findings**: What informs the content
- **Dependencies**: What they need from other specialists or user

## Non-Coding Plan (Step 1F)

Break work into deliverable sections. Each section:
- Self-contained part of final deliverable
- Reviewable independently
- Clear specialist ownership

Specify per section: specialist (with equipped skill), scope, inputs, output format, review criteria.

## Non-Coding Delivery (Step 5)

1. Compile final deliverable (assemble approved sections, apply formatting)
2. Present to user with file paths
3. Execute external actions (bookings, sends) with user confirmation per action
4. Handoff: summarize what was delivered, note follow-up actions
