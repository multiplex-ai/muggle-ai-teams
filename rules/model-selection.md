# Model Selection Strategy

**Opus** — Use for anything requiring thinking, reasoning, or deduction:
- Bug fixes and debugging (root cause analysis)
- Architectural decisions and system design
- Complex reasoning and deduction tasks
- Code review (3-pass reviewer)
- Planning and research phases
- Security analysis

**Sonnet** — Use for main development work:
- Feature implementation
- Orchestrating multi-agent workflows
- Standard coding tasks
- Build error resolution
- E2E test generation

**Haiku** — Use for lightweight, high-frequency tasks:
- Documentation updates and lookups
- Simple code generation and formatting
- Worker agents in multi-agent pipelines
- Boilerplate scaffolding
